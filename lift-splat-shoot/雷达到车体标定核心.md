## lidar2car 标定主要思路
```
1. 项目地址位于：https://github.com/PJLab-ADG/SensorsCalibration/tree/master/SensorX2car/lidar2car

2. 标定涉及的旋转角度也就是包含分别绕x\y\z三个轴的旋转角度，其中绕xy的旋转主要是通过提取地面点云构造地面法向量之后与虚拟Z轴之间的关系从而得出的，而绕Z轴的角度主要是通过比较IMU时刻记录的车体姿态与轨迹切线之间的夹角得出，项目主要难点在地面点云的提取与平面法向量的构建
```
```
提取地面点的三种方法记录

方法1：
bool GroundExtractor::LPRFitting(const PointCloudPtr in_cloud,
                                 PointCloudPtr g_cloud,
                                 PointCloudPtr ng_cloud,
                                 PlaneParam * plane) 
    MaxRangeFilter（）去除距离过大的点
    sort（）根据Z轴高度排序
    循环去除明显异常的低点（噪声）
        （如果计算出的下一个点的z轴距离与当前点Z轴距离之差大于0.2米，说明当前点为噪声点并删除）
    根据z轴排序后的点云，计算前%-的点的平均高度，作为地面高度阈值，并根据该阈值将点云分为地面点和非地面点
    迭代优化
        （1）FittingPlaneMesh(g_cloud, plane)使用最小二乘法拟合平面参数
        （2）清空地面和非地面点云，根据点到上面拟合平面距离重新分割地面/非地面点
        （3）计算当前平面与上一次平面之间各个参数之间的差值
        （4）法向量检查，确保Z轴向上

方法2：
bool GroundExtractor::RandomRansacFitting(const PointCloudPtr in_cloud,
                                          PointCloudPtr g_cloud,
                                          PointCloudPtr ng_cloud,
                                          PlaneParam* plane) 
    外层循环（多次RANSAC尝试）
        内层循环（内层是当前轮中尝试多少种随机三点组合）
            从原始点云里面随机选取三个点，计算平面参数
            计算内点数量
            更新最佳平面参数
    RandomSearchPlane（） 随机搜索优化平面参数（粗调、精调），对当前最佳平面进行微调搜索
    更新最佳平面参数
    重新提取该最佳平面的内点作为地面候选点
    FittingPlaneMesh(g_cloud, &best_plane);使用地面候选点拟合平面，获得更精确的平面参数
    重新获取地面点和非地面点，并保证法向量是正的   

方法三：
bool RPCalib::GroundPlaneExtraction(const PointCloudPtr &in_cloud,
                                    PointCloudPtr g_cloud,
                                    PointCloudPtr ng_cloud, PlaneParam &plane)
    使用PCL库的标准实现、本质还是RANSAC
    初始化PCL对象（配置RANSAC参数）
    执行RANSAC分割（随机采样 → 模型拟合 → 内点统计 → 模型优化）
    提取地面点和非地面点
    转换平面参数格式
    输出结果
```
